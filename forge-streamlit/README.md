# oTree Forge Studio（Streamlit版）

ESL仕様をGUIで編集し，oTreeプロジェクトの生成・bot検証・書き出しまで画面内で行う．
生成エンジンは forge.py（React版とは異なりPython単一実装）．

## ローカル / 研究室サーバで動かす
```
pip install -r requirements.txt
streamlit run streamlit_app.py --server.fileWatcherType poll
```
ブラウザで http://localhost:8501 を開く．Tailscale経由のサーバに置けば手元のどの端末からも使える．
（`--server.fileWatcherType poll` は macOS + 新しいPythonで標準のファイル監視
（fsevents）がクラッシュする問題の回避策である．通常運用でも付けて損はない）

## Streamlit Community Cloud で公開する
1. このフォルダをGitHubリポジトリにpush
2. https://share.streamlit.io で New app → リポジトリと streamlit_app.py を指定
3. AI機能を使う場合は App settings → Secrets に
   `ANTHROPIC_API_KEY = "sk-..."` を設定（コードには絶対書かない）
4. botテストを使う場合は，アプリ右下の「oTree実行環境をセットアップ」を一度押す
   （専用venvにoTreeをインストールする．コンテナ再起動後は再実行が必要）

## 依存関係の設計（重要）
StreamlitとoTreeは最新版同士では共存できない（starlette / websockets の衝突）．
このため，**メイン環境（requirements.txt）にはoTreeを入れない**．
`otree test` は otree_runner.py が管理する専用venv（既定 `~/.otree_forge_env`，
環境変数 `OTREE_FORGE_ENV` で変更可）内のotreeコマンドで実行する．
この分離により，Streamlitは常に最新版へ更新できる．
専用venvが無い場合はPATH上のotreeにフォールバックする．

## 機能
- プリセット（白紙・独裁者・信頼・公共財・アンケート＝入力形式の見本）
- **複数app対応**：実験本体→アンケート等，appを追加・削除・並べ替えできる
  （app_sequence は自動で同期される）
- **ブロックパレット**：説明・同意・数値入力・リッカート3問・WTP測定・
  待機＋利得計算・結果・結果グラフの定型ブロックをワンクリックで追加できる
  （ページ＋フィールド＋ロジックの一式が挿入され，名前は自動で重複回避）
- **実験フロー図**：全appのpage_sequenceを図で俯瞰できる（ページ種別を色分けし，
  form欄・利得計算フック・表示条件の分岐を注記する）
- **デザインテーマ**：フォント（Google Fonts）・基調色・背景色・文字色・文字サイズを
  GUIで選ぶと，全ページ共通の `_templates/global/Page.html` を自動生成して一括適用する
- **参加者画面プレビュー**：右ペインでページを選ぶと，参加者に見える画面を
  その場で描画する（テーマ・カスタムwidget込み．デモサーバの起動は不要）．
  `{{ C.X }}` 等は定数・test_value のサンプル値で置換する近似表示である
  （通貨値は use_points・通貨コードに応じてoTree実機に近い書式で表示する）
- **セクションエディタ**：ページ本文をHTMLを書かずに組める．見出し・段落・
  強調ボックス・画像・表・2カラム・フォーム欄・グラフ・生HTMLのセクションを
  並べ替えて配置する（フォーム欄を本文の途中に置ける）．ESLには `content`
  配列として保存され，「HTML直書き」モードとはワンクリックで相互変換できる
- **グラフページ（js_vars）**：グラフセクションの「データ源」を選ぶだけで
  js_vars（Pythonコード）を自動生成する（メンバー別の利得／メンバー別の入力値／
  ラウンド推移．カスタムを選べば手書きもできる）．本文の `<script>` からは
  `js_vars.キー名` でデータにアクセスできる．Chart.js による「結果グラフ」
  ブロックも同梱
- **定番スニペット**：HTMLセクションにはYouTube埋め込み・音声・大きな数字・
  折りたたみ・チャット・ライブ通信雛形等を，js_vars欄には定番パターンを
  ワンクリックで挿入できる
- **ライブページ（リアルタイム通信）**：ページに live_method（Python）を書くと，
  本文の `liveSend(データ)` がサーバに届き，返り値が `liveRecv(data)` に配られる
  （チャット・市場・投票集計など）．「⚡ ライブページ」ブロックで動く見本を
  挿入できる．live_test を書けば botテスト中の liveSend も模擬できる
  （`call_live_method` を自動生成）．グループチャットだけなら本文に
  `{{ chat }}` と書くだけでよい（「💬 チャット」ブロック）
- **到着順グループ化（group_by_arrival_time）**：先頭の待機ページでONにすると，
  到着した人から順に組を作る（オンライン実験で全員の同時開始を待たなくてよい）
- **部屋（rooms）**：固定URL `http://サーバ/room/部屋名` で参加者を受け入れる本番用の
  入口を作れる．参加者ラベル（座席番号・ID等）を入れると `_rooms/部屋名.txt` を
  自動生成し，出席確認と席の対応づけができる．セキュアURL（ラベルごとに推測不能な
  URLを発行，成りすまし防止）にも対応．デモサーバ起動時には部屋URLも表示する
- **undo/redo**：サイドバーの「↩ 元に戻す／↪ やり直す」で直近80操作まで戻せる
- **静的診断（lint）**：生成は可能だが実セッションで事故になりやすい構成
  （表示条件＋制限時間なしのページの直後の待機＝デッドロックの芽，
  グループなしでの待機ページ，連続する待機ページ等）を右ペインで警告する
- グループなし（PLAYERS_PER_GROUP=None）の個人課題・アンケートに対応
- ページごとの制限時間（timeout_seconds）をGUIから設定できる
- **豊富な入力形式**：数値欄・テキスト欄・ドロップダウン・ラジオボタン（縦/横）・
  チェックボックスに加え，以下の独自形式をテンプレートHTML/JSとして自動生成する
  - スライダー（値をライブ表示，両端ラベルつき）
  - つまみ非表示スライダー（クリックするまで未回答扱い．アンカリング回避，WTP測定向き）
  - ボタン選択（押した瞬間に送信して次ページへ）
  - 星評価（クリックで選択）
  - リッカート表（同じ選択肢の連続質問を1つの表にまとめる）
  - 数値ステッパー（−/＋ボタン）
  - 複数選択チェックボックス（カンマ区切りでstr型に保存）
  - これらで足りない場合はページ単位の「HTML完全手動」モードで自由に記述できる
- 基本設定・定数・モデル（表形式編集）・ロジック（スニペット挿入）・ページ（表示条件ビルダー）
- 右ペイン：仕様検証，bot導出可否，生成 __init__.py / ESL JSON のライブプレビュー
- 仕様検証の内容：Python識別子・型・重複・oTree予約語の検査，min/max/display_if 等の
  式とlogic・vars_for_template・js_vars・live_method 本体の構文チェック，
  test_value の範囲検査，session設定の整合性検査（players_per_group は None か2以上，
  デモ参加者数がグループ人数で割り切れるか等），theme の色形式・文字サイズ検査，
  group_by_arrival_time の配置検査（先頭の待機ページのみ）
- 書き出し：ESL仕様（JSON），oTreeプロジェクト一式（zip，esl_spec.json同梱）
- **otree test の画面内実行**：botが実験を自動プレイし，合否とログを表示する
  （専用venvはアプリ内ボタンから自動構築）
- **botの行動検証**：どの参加者がどのページで何を送信したか（行動ログ）と，
  プレイ結果のデータ（入力値・計算された利得，1行＝1人×1ラウンド）を表で表示する．
  利得計算ロジックが意図通りかをここで確認できる
- **デモプレイ**：現在の仕様で `otree devserver` を起動し，実際の参加者画面を
  ブラウザで開いてプレイできる（http://localhost:8503/demo ．
  ローカル・研究室サーバ向け．Streamlit Community Cloudでは外部から届かない）．
  起動時に前回の孤児プロセス・一時ディレクトリを自動で片付ける
- AI仕様生成（サイドバー）：日本語の指示で仕様を生成・修正．
  生成結果は forge.validate で自動検証し，エラーはAIに差し戻して自動修正する（最大3回）
